## Ambiguity 1: 要約機能の MVP スコープ

**検出した不整合**:
- Q1 で「**その日の主要なニュースを要約して短時間で概要を把握できるようにする**」と明示されている

つまり「要約機能は MVP に含めるのか、それとも将来フェーズに回すのか」が曖昧です。

### Clarification Question 1

D) **MVP では「タイトル + 元記事の冒頭抜粋 (RSS の description フィールド等)」を表示するに留める** — LLM での再要約は行わず、ソース由来の要約をそのまま使う

## Ambiguity 2: X (Twitter) からの情報取得方法

**検出した不整合**:
- Q1 で取得対象に **X (Twitter)** が含まれている
- Q4 で `A) RSS フィード` および `B) 外部ニュース API` を選択
- しかし X (Twitter) は **公式 RSS フィードを廃止** しており、X 公式 API も **有料プランが必須** (基本枠は月額数百ドル〜)。「RSS / 外部ニュース API」のままでは X からの収集は実現困難です。

### Clarification Question 2
X (Twitter) からの情報取得は、以下のうちどう扱いますか?

A) **MVP のスコープから X (Twitter) を除外する** — Zenn・はてなブックマーク・一般ニュース (RSS / NewsAPI 等) のみを対象に、まず確実に動く範囲で着手する
B) **X 公式 API を有料利用する** — Basic/Pro プランを契約 (月額コストを許容)
C) **サードパーティのミラー / RSS ブリッジサービスを利用する** — 例: nitter インスタンス、rss-bridge、RSSHub 等の非公式サービス (動作不安定リスクを許容)
D) **特定アカウントを監視するのみで、Web スクレイピングする** — 利用規約上のリスクと取得の不安定さを許容
X) その他 (please describe after [Answer]: tag below)

[Answer]: X
Xの代わりに、Togetterをスクレイピングして情報収集する

## Ambiguity 3: 記事詳細ページの扱い

**検出した不整合**:
- Q3 (MVP 機能) で `A) ニュース記事一覧ページ` のみを選択し、`B) ニュース記事の個別詳細ページ` は選択されていない
- 一方、Q1 のプロダクト像では「収集したニュースを一覧で見て、要約で概要を把握する」となっている

詳細ページが無い場合、ユーザー (= umatoma さん) は一覧から **どこで本文や元記事に到達するのか** が未確定です。

### Clarification Question 3
記事詳細の閲覧体験はどうしたいですか?

A) **一覧ページから元記事 (Zenn / はてブ / 一般ニュース等の外部 URL) に直接遷移する** — 自サイト内には詳細ページを設けない (シンプル)
B) **自サイト内に簡易詳細ページを用意する** — タイトル + 要約 + 元記事へのリンク程度の最小構成
C) **自サイト内に本格的な詳細ページを用意する** — 全文取得 (記事抜粋・OGP 画像等) + 元記事リンク
D) **MVP では A の方針 (外部 URL に直接遷移)、第 2 フェーズで B / C に拡張する**
X) その他 (please describe after [Answer]: tag below)

[Answer]: A
